# MiniGCS

MiniGCS 是一个模块化设计的项目，提供了多种功能模块和测试组件。以下是项目的整体结构和描述：

## 目录结构

### 1. `Inc`
包含项目的头文件，分为以下模块：
- **AirLine**: 航线管理相关的头文件。
- **Link**: 链接管理模块，包括 `Private` 子模块，用于处理内部链接逻辑。
- **Plat**: 平台接口模块，也包含 `Private` 子模块，定义与操作平台相关的接口功能。
- **Extern**: 外部接口的头文件。
- **Private**: 私有模块的头文件。

### 2. `Src`
包含项目的源代码，与 `Inc` 中的头文件一一对应：
- **AirLine**: 航线管理的具体实现。
- **Link**: 链接管理模块的实现，包括 `Private` 子模块。
- **Plat**: 平台相关功能的实现，包括 `Private` 子模块。
- **Extern**: 外部接口的具体实现。
- **Private**: 私有模块的实现。

### 3. `Test`
测试模块。
- **qml**: 包含与界面或可视化相关的测试文件。

### 4. `.git`
Git 版本控制目录，用于管理项目的代码历史和协作。

## 功能描述

1. **航线管理（AirLine）**: 提供航线规划、信息管理及相关操作的功能模块。
2. **链接管理（Link）**: 处理设备连接、通信协议及相关逻辑。
3. **平台接口（Plat）**: 封装与操作平台交互的接口。
4. **外部接口（Extern）**: 提供与其他系统集成的方法和接口。
5. **私有模块（Private）**: 处理内部功能，不直接暴露给其他模块。

## 使用说明
本项目的代码模块清晰分层，建议按照以下步骤使用：
1. 根据 `Inc` 和 `Src` 的模块功能，找到需要的头文件和对应实现。
2. 使用 `Test` 模块进行单元测试或集成测试。
3. 配置 Git 仓库，方便版本管理。
